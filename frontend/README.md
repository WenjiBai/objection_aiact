# OBJECTION! AI ACT React frontend

A React + Vite rewrite of the Streamlit UI in `../app.py` + `../ui/`.

The UI is wired to the Python courtroom backend through `../backend/http_api.py`.
The demo-case buttons still load the local Case A / Case B mocks instantly, but
clicking "Open courtroom hearing" posts uploads / pasted text to
`run_courtroom()` and renders the returned `CaseFile`.

## Run

```bash
cd frontend
npm install
npm.cmd run serve
```

Then open `http://127.0.0.1:8787`.

For frontend hot-reload development, run two terminals instead:

```bash
cd frontend
npm.cmd run api
npm.cmd run dev
```

## Backend Wiring

- `POST /api/run-courtroom` builds a `CaseFile`, calls `run_courtroom()`, and returns the completed case.
- `POST /api/chat` calls `handle_chat()` for cross-examination turns.
- `POST /api/export-pdf` returns the structured Preliminary Verdict PDF generated by `shared.pdf_export`.
- Vite proxies `/api/*` to `http://127.0.0.1:8787`.
- In single-server mode, `backend/http_api.py` also serves `frontend/dist`.

## What's Included

- Sidebar with dark/light toggle and demo-case buttons.
- Landing hero, drag-drop file zone, paste-text area, demo case quick start.
- Case banner with sticky tier pill.
- 6-agent stepper with running/completed states.
- Verdict card with SVG confidence ring.
- Evidence Board, Symbolic Risk Gate, OBJECTION cards, governance checklist.
- Cross-examine chat backed by `handle_chat()` when the API server is running.

## What's Not Included

- AI Act corpus / Guide / Search cases are static frontend dialogs.
